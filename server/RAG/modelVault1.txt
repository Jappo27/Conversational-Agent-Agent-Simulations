Conversational Agent-Agent Simulations API Project Plan 52209404 Oliver Japp u02oj22@abdn.ac.uk Department of Computing Science, University of Aberdeen, Aberdeen AB24 3UE, UK Introduction Psychologists often spend a significant portion of their time conducting interviews with subjects [1].When conducting surveys, human participants should be the gold standard when sourcing reliable information.However, when conducting these interviews, difficul- ties can occur [2] as it can face a myriad of issues, such as long interview processes, thus limiting samples, issues with ethics, or a lack of participants [3][4][5][6].High-quality, reliable data is required to generate reliable insights into groups of people or small demo- graphics [7].RAG (Retrieval-Augmented Generation) agents are a recent development.RAG is a framework introduced in 2020 [8] that allows more recent data to be incorporated into an LLM.
RAG agents can be decomposed into three stages: accessing a database with sources selected by the user, augmenting and altering the prompt to ensure the data align with the retrieved information, and generating the new, updated prompt [9]. Since LLMs rely on their training data, which can become out-of-date within a short period, RAG agents enable the most current data to be incorporated into an LLM’s response [10].These RAG agents will utilise custom functions to ensure they remain stable.These functions include context cleansing, information retrieval, and goal perception [11].Within this project, RAG agents will be customised in several stages: Prompt, Data, and Ollama Skeleton.Prompts will be customised before being fed to the agent, allowing the researcher to detail the specific persona traits.[12] The data will consist of documents such as policy, procedure and other relevant non-sensitive documents.
The Ollama skele- ton will be used to interface with Ollama, an open-source tool for running LLMs locally [13]. The Ollama elements will include features such as TOP K, Seed, temperature, etc.This allows users to customise each aspect of the RAG agent fully.These agents will then engage in agent-to-agent conversations to generate a large volume of synthetic data.The next step is to utilise Natural Language Processing (NLP) [14] and compare the synthetic data with real conversational data.This comparison will examine the distribution of words in each response and each conversation, then document 1 deviations from the norm [15] [16].This project is a collaboration between the University of Aberdeen’s Psychology and Computer Science departments on Human AI interaction.The aim is to develop a method- ology to simulate two RAG agents producing conversational data, then use NLP to com- pare synthetic vs.
real conversations to demonstrate that agents remain stable and con- duct conversations in alignment with their prompt. Goals This project aims to develop a tool for psychology researchers that allows them to generate conversational transcripts between two RAG agents.This will allow them to reduce the number of real individuals required for a study by utilising synthetic data.•The project aims to accomplish: Provide an interface for designing prompts, and Ollama settings to customise the AI agents.Then utilise these prompts, and settings to build RAG agents and generate conversational transcripts with them.•The project aims to measure: The variance between synthetic and real conversational data, using NLP.These met- rics will be measured by utilising NLP to assess the differences in natural language between real and synthetic conversations.This will be conducted by measuring the deviation of the synthetic conversations from the norm.
•The project aims to prove: That AI agents can be assigned custom prompts and remain stable. Stable in this context is defined as: Non-repetition of phrases, Contextual awareness (Both utilis- ing provided information when appropriate and not hallucinating information when not provided), and responding in a manner consistent with the assigned prompts.•The project may extend to include: The analysis of RAG agent and real conversational transcripts using NLP tone analysis, the analysis of outliers using LLM as a judge, and the construction of a methodology to allow for two or more agents in a turn-based order.Methodology •Reading about related work - Both to understand established practices and gather potential resources for implementation.•Develop a user interface for prompt design and Ollama skeleton configuration, that allows for maximum control for researchers.
2 •Develop a RAG methodology: This includes incorporating all previous steps and developing methods for an agent to call that would increase stability. •Generate the data utilising two RAG agents in conversation •use NLP to analyse the deviation from synthetic and natural conversations Resources Required The resources required for this project are Ollama, an open-source software; a down- loadable Ollama model in the range of 8 billion parameters; and the Python and React programming languages.The developed software will require a system capable of gener- ating large volumes of data.The researcher has access to such a system but is limited to running models with 10 billion parameters.Some resources that may prove more difficult to obtain include additional RAM sticks, as the cost of RAM has increased dramatically.This could be mitigated by using the£300 Azure personal student credit for cloud computing or by accessing university-provided services.
If the required processing power is not available from either party, a primary solution would be to downscale the agent parameter count to ensure it can run on available hardware or increase the processing time. A significant consideration is that this project will require ethical approval to access sensitive documents related to policy, conversations, and personal details, to allow for NLP analysis and comparison of agents and their variation from real conversations.If these documents cannot be procured or ethical approval is not granted, the main alternative is to conduct interviews to generate these conversations on non-sensitive topics and to update the supporting RAG documents to reflect those topics.Once we generate and analyse this data, we could then extrapolate the results to conclude the potential of the intended study.
Risk Assessment When considering potential risks, this project’s main concerns are data leakage or con- fidentiality issues when working with LLMs, and the potential for prompt injection. A potential mitigation is to ensure that the final product is not shipped containing any doc- ument in the agent’s RAG database that can directly identify a user and fully anonymise data whilst in production.Potential future risks and impacts of this work include the ethical implications of synthetic data in psychology.The purpose of this project is to develop a methodology that allows psychologists to produce synthetic conversations between two groups on a large scale.The risk lies in reducing the number of real samples and extrapolating conclusions without a proper basis.These risks can be mitigated by requiring researchers who intend to use this software to both declare its use and the resulting synthetic data and to source an appropriate number of real subjects.

Our goal is to empower users to generate larger volumes of datasets without requiring unique scripts for each instance. The purpose of this study is to get feedback from our target demographic - researchers- and see if the application is helpful in streamlining and simplification generating large datasets, along with gathering potential next steps.Our aim is to see if our application has achieved the scope that was initially set and potential next steps for the project.Why have I been invited to take part?You have been invited to participate in this as you are a part of the University of Aberdeen NCS department.Do I have to take part?Participation in this research study is completely voluntary.This participant information sheet has been produced to help you decide if you would like to take part in this study.It is up to you (and you alone) to decide whether you wish to take part.

The data collected will include user opinions on ease of use, functionality adequacy, performance, reliability, and suggestions for improvement.
